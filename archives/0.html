git命令

命令 作用
git config --global user.name 用户名 设置用户签名
git config --global user.email 邮箱 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件 将文件加入暂存区
git rm --cached文件 将文件从暂存区移除
git commit -m 备注 文件 将暂存区的文件提交到本地库
git reflog(git log) 查看版本信息
git reset --bard 版本号 版本穿梭

设置用户签名只是为了用于区分，与之后登录github没有任何关系

git add 文件(未被追踪的文件)

文件被修改后需先add加入暂存区，在通过commit提交到本地库形成新的版本。

git分支

分支可以理解为副本，一个分支就是一个单独的副本

命令 作用
git branch 分支名 创建分支
git branch -v 查看分支
git checkout 分支名 切换分支
git merge 分支名 把指定的分支合并到当前分支上
git branch -d 分支名 删除分支

正常合并通过merge命令即可，但产生冲突时，则不能简单的通过该方法进行合并，当后面状态出现merging时，表明产生冲突。冲突产生的原语为合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改，git无法决定使用哪一个，必须人为决定新代码内容。

修改方式：人为进入文件进行修改，修改完成后进行add commit，commit不能带文件名。

git团队协作

团队内协作：push->clone->push->pull

跨团队协作：fork(在我的远程库中创建一个分支)->clone->push->pull request->审核->merge->pull

github操作

命令 作用
git remote -v 查看别名
git remote add 别名 远程地址 创建别名
git push 别名 分支 推送本地分支到远程库
